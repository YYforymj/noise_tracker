# Noise Monitor 噪声监测 — 产品说明文档

> 一个移动端优先的噪声检测 Web 应用，利用手机或电脑麦克风实时测量环境噪音等级。

---

## 1. 产品定位

Noise Monitor 是一个**纯前端**的噪声检测工具，无需下载安装，打开浏览器即可使用。核心场景：

- 📱 在嘈杂环境中快速了解当前噪音水平
- 📊 记录历史噪音数据，查看趋势变化
- 🌙/☀️ 支持深色和浅色主题，中文和英文界面
- 🔒 所有数据存储在浏览器本地，不上传任何服务器

**目标用户**：需要在日常生活或工作中了解环境噪音水平的人（如：居家办公者、教师、家长、声学爱好者等）

## 4. 交互流程

### 4.1 首次使用

```
用户打开页面
    ↓
显示默认界面（分贝显示 0，等级"开始监测"，统计全为 0）
    ↓
用户点击「开始监测」
    ↓
浏览器弹出麦克风权限请求
    ├── 允许 → 开始实时采集
    │     ↓
    │   分贝数值实时更新，趋势图开始绘制
    │   历史记录显示之前保存的会话
    │
    └── 拒绝 → 显示红色错误提示条
           "麦克风权限被拒绝，请在浏览器设置中允许麦克风访问"
```

### 4.2 监测过程

```
正在监测中
    ↓
用户可操作：
├── 点击「暂停监测」→ 暂停采集，按钮变为「继续监测」
│     ↓
│   点击「继续监测」→ 恢复采集，不需要重新请求麦克风
│
├── 点击「■」停止按钮 → 保存会话到历史记录，停止监测
│     ↓
│   回到初始界面，历史记录列表更新
│
└── 切换到其他标签页 → 自动暂停采集，切回后自动恢复
```

### 4.3 主题和语言切换

```
点击 ☀️/🌙 图标 → 立即切换深色/浅色主题，刷新后保持选择
点击 中/EN 按钮 → 立即切换中文/英文，刷新后保持选择
```

## 5. 分贝与等级说明

### 5.1 显示原理

应用使用 **Web Audio API** 从麦克风采集音频数据，计算 **RMS（均方根）** 并转换为 **dBFS（分贝满刻度）**，然后映射到 **0-120 相对范围**供用户参考。

> ⚠️ **重要**：显示的分贝值是**相对噪音等级**，不是专业声级计的 SPL（声压级）校准值。实际分贝值取决于设备麦克风灵敏度、距离声源远近等因素。

### 5.2 等级对照

| 等级 | 相对分贝 | 大致对应环境 | 中文 | English |
|------|----------|-------------|------|---------|
| 🟢 安静 | 0-50 | 图书馆、卧室、安静的办公室 | 安静 | Quiet |
| 🟡 适中 | 50-70 | 正常对话、办公室、餐厅 | 适中 | Moderate |
| 🟠 嘈杂 | 70-85 | 繁忙街道、吸尘器、洗碗机 | 嘈杂 | Loud |
| 🔴 危险 | 85+ | 施工现场、演唱会、工厂车间 | 危险 | Danger |

### 5.3 数值平滑

为避免数字快速跳动影响阅读，应用使用**指数平滑算法**：
- 上升时（声音变大）：attack 系数 0.06，响应较慢
- 下降时（声音变小）：release 系数 0.02，衰减较慢
- UI 每 1 秒更新一次显示数值

## 7. 数据与隐私

### 7.1 数据存储位置

所有数据存储在**用户浏览器本地**：

| 存储方式 | 内容 | 保留期 |
|---------|------|--------|
| localStorage | 主题偏好、语言偏好 | 永久（除非用户清除） |
| IndexedDB | 监测会话、实时读数 | 会话永久保留，读数 30 天自动清理 |

### 7.2 数据传输

**不传输任何数据到外部服务器。** 所有计算和处理均在用户设备本地完成。

### 7.3 麦克风权限

- 麦克风仅在用户主动点击「开始监测」后请求
- 暂停期间不采集音频
- 页面隐藏时自动暂停采集
- 仅计算分贝数值，**不录制、不保存、不传输音频内容**

|--------|------|------|

| Edge 79+ | ✅ 完全支持 | 基于 Chromium |
| IE | ❌ 不支持 | |

### 安全上下文要求

麦克风 API (`getUserMedia`) 需要**安全上下文**：
- ✅ `https://` 域名
- ✅ `http://localhost`
- ❌ `http://192.168.x.x`（非 localhost 的 HTTP 地址）

如果在不安全的环境下打开，页面会显示提示："麦克风需要安全连接(HTTPS)才能使用"。

## 9. 部署要求

### 最低要求
- 任何静态文件服务器（Nginx、Apache、Vercel、Netlify 等）
- 必须配置 HTTPS（麦克风 API 要求）
- SPA 路由需配置 fallback 到 `index.html`

### 依赖的外部资源
- Google Fonts（Nunito + JetBrains Mono）—— 需要外网访问
- 无其他外部 CDN 或 API 依赖
